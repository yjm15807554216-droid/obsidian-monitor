# 复盘：V3 分位数告警噪音问题 (2026-04-08)

> 返回首页：[[00_告警追踪系统_首页]]

## 问题

2026-04-07 下午 15:00 ~ 20:00 的 5 小时内，V3 分位数告警连续触发 **23 条**。其中大量告警对应的控筹率波动极小，不具备实际交易参考价值，属于**噪音告警**。

Telegram 被刷屏，严重影响了对真实信号的辨识。

## 数据分析

### 23 条告警按币种分布

| 币种 | 告警数 | 控筹率范围 | spread (max-median) | 是否噪音 |
|------|--------|-----------|---------------------|---------|
| DOTUSDT | 5 条 | 1.98%-2.05% | 0.187pp | ✅ 噪音 |
| EDGEUSDT | 4 条 | 5.76%-6.11% | 0.498pp | ✅ 噪音 |
| MUSDT | 3 条 | 0.48%-0.49% | **0.018pp** | ✅ 极端噪音 |
| AAVEUSDT | 3 条 | 3.13%-3.18% | 0.038pp | ✅ 噪音 |
| ZECUSDT | 2 条 | 3.14%-3.35% | 0.170pp | ✅ 噪音 |
| NEARUSDT | 2 条 | 4.10%-4.12% | 0.086pp | ✅ 噪音 |
| SUPERUSDT | 1 条 | 7.47% | 0.536pp | ✅ 噪音 |
| FILUSDT | 1 条 | 5.54% | 0.132pp | ✅ 噪音 |
| **REDUSDT** | **2 条** | **17.44%-17.75%** | **1.687pp** | **❌ 真信号** |

### 关键发现

23 条告警中只有 **2 条是真信号**（REDUSDT），其余 **21 条全是噪音**。噪音率 **91%**。

### 噪音的根本原因

V3 分位数告警的逻辑是：

```
如果 当前控筹率 > 过去 14 天数据的 95% 分位 → 触发告警
```

这个逻辑有一个**隐含假设**：95% 分位代表"异常高"。

但对于控筹率波动极小的币，这个假设不成立：

**MUSDT 典型案例：**
- 14 天控筹率范围：0.46% ~ 0.49%
- 中位数：0.48%
- 95 分位：0.48%
- 当前值：0.49%（比中位数高了 0.01 个百分点）
- 分位数：95.1% → 触发告警

数学上确实在 95% 分位以上，但实际意义为零。0.01pp 的波动在市场上根本不值一提。

### 另一个问题：冷却期太短

即使触发了，同一个币在 1 小时冷却期后又反复触发：
- DOTUSDT 在 15:19、16:22、17:23、17:23、20:10 分别触发（5 次）
- EDGEUSDT 在 15:16、16:17、17:17、18:17 分别触发（4 次）

说明 1 小时冷却对分位数告警来说太短。

## 解决方案

### 已实施：PERCENTILE_MIN_SPREAD 过滤器

在 `process_percentile_alert` 函数中增加判断：

```python
spread = stats['max'] - stats['median']
if spread < config.PERCENTILE_MIN_SPREAD:
    return False  # 波动太小，跳过
```

配置：
```python
PERCENTILE_MIN_SPREAD = 0.01  # 1 个百分点
```

### 过滤效果验证

| 币种 | spread | 过滤结果 |
|------|--------|---------|
| MUSDT | 0.018pp | ❌ 被过滤 |
| AAVEUSDT | 0.038pp | ❌ 被过滤 |
| NEARUSDT | 0.086pp | ❌ 被过滤 |
| FILUSDT | 0.132pp | ❌ 被过滤 |
| ZECUSDT | 0.170pp | ❌ 被过滤 |
| DOTUSDT | 0.187pp | ❌ 被过滤 |
| EDGEUSDT | 0.498pp | ❌ 被过滤 |
| SUPERUSDT | 0.536pp | ❌ 被过滤 |
| **REDUSDT** | **1.687pp** | **✅ 保留** |

23 条告警中 21 条被过滤，仅保留 2 条 REDUSDT 真信号。**噪音削减 91%**。

## 后续观察

### 需要关注的风险

1. **阈值是否合理**：1pp 是经验值。如果未来出现 spread 在 0.5-1pp 之间但确实有意义的信号，需要调低阈值。
2. **高位回调场景**：如果一个币长期在高控筹率运行（比如 median 已经是 20%），那 max-median 的 spread 可能很小，但绝对值其实很高。当前 V3 分位数告警只看 ≥95% 高位，所以暂时不受影响。
3. **冷却期调整**：分位数告警的冷却期仍然是 1 小时（`RATE_ALERT_COOLDOWN = 3600`），如果噪音过滤效果不够好，可以考虑延长到 4 小时。

### 结论

- V3 分位数告警的核心价值在于"控筹率突破自身历史极值"
- 但"极值"需要有绝对意义，不能是统计噪音
- PERCENTILE_MIN_SPREAD 是一个简单有效的过滤手段
- 真正的高质量信号应该由 V4 复合告警来捕获（三条件 AND）
